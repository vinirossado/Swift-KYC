# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| latest  | Yes       |
| < latest | No       |

Only the most recent release receives security updates. We recommend upgrading promptly when a new version is available.

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

If you discover a security issue in IdentityKit, we appreciate your responsible disclosure. Please report it by emailing:

**security@identitykit.dev**

Include as much of the following as possible:

- A description of the vulnerability and its potential impact.
- Steps to reproduce or a proof of concept.
- The version(s) of IdentityKit affected.
- Any suggested remediation, if you have one.

## What to Expect

- **Acknowledgement** within 2 business days confirming we received your report.
- **Initial assessment** within 5 business days with our evaluation of severity and scope.
- **Resolution timeline** communicated once the issue is triaged. Critical vulnerabilities are prioritised for the next patch release.
- **Credit** in the release notes (unless you prefer to remain anonymous).

## Scope

This policy covers the IdentityKit Swift package and its published modules:

- IdentityKitCore
- IdentityKitCapture
- IdentityKitUI
- IdentityKitNetwork
- IdentityKitStorage

The demo application (`DemoApp/`) is provided for reference only and is not covered under this policy.

## Disclosure Policy

We follow coordinated disclosure. We will work with you to agree on a disclosure date, typically within 90 days of the initial report. We will not take legal action against researchers who follow this policy in good faith.
